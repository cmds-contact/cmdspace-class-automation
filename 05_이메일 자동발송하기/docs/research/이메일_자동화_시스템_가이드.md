## 1. 개요

### 이메일 자동화란?

이메일 자동화는 특정 조건이나 이벤트에 따라 자동으로 이메일을 발송하는 시스템입니다.

### 이메일 유형

| 유형 | 설명 | 예시 |
|------|------|------|
| **트랜잭션 이메일** | 사용자 행동에 의해 즉시 발송 | 결제 확인서, 주문 알림, 비밀번호 재설정 |
| **마케팅 이메일** | 정기적/캠페인 기반 발송 | 뉴스레터, 프로모션, 이벤트 안내 |

### 구현 방식 3가지

```
┌─────────────────────────────────────────────────────────────┐
│                    이메일 자동화 구현 방식                      │
├─────────────────┬─────────────────┬─────────────────────────┤
│   직접 구축      │   API 서비스     │    노코드 도구           │
│  (Python 등)    │  (SendGrid 등)  │   (Zapier, Make 등)     │
├─────────────────┼─────────────────┼─────────────────────────┤
│ 완전한 통제      │ 균형잡힌 선택    │ 가장 쉬운 시작           │
│ 높은 기술 요구   │ 적절한 기술 요구  │ 기술 불필요             │
│ 최저 비용       │ 합리적 비용      │ 상대적 고비용            │
└─────────────────┴─────────────────┴─────────────────────────┘
```

## 2. 핵심 구성요소

### 2.1 필수 요소

| 구성요소 | 설명 | 중요도 |
|----------|------|--------|
| **발송 인프라** | SMTP 서버 또는 API | ⭐⭐⭐ |
| **템플릿 시스템** | HTML 이메일 디자인 | ⭐⭐ |
| **수신자 관리** | 구독자 목록, 세그먼트 | ⭐⭐ |
| **인증 설정** | SPF, DKIM, DMARC | ⭐⭐⭐ |
| **통계/분석** | 오픈율, 클릭률 추적 | ⭐⭐ |

### 2.2 이메일 인증 (2024년 필수)

> ⚠️ **중요**: 2024년 2월부터 Google, Yahoo는 일 5,000건 이상 발송 시 SPF, DKIM, DMARC 미설정 이메일을 스팸 처리합니다.

```
SPF  : 발송 서버 IP 인증
DKIM : 이메일 서명 인증
DMARC: SPF/DKIM 정책 설정
```

## 3. 서비스 유형별 분류

### 3.1 API 기반 이메일 서비스

개발자가 API를 통해 이메일을 발송하는 서비스입니다.

| 서비스 | 특징 | 월 10,000건 비용 |
|--------|------|-----------------|
| [AWS SES](https://aws.amazon.com/ses/) | 최저가, AWS 생태계 | ~$1 |
| [SendGrid](https://sendgrid.com/) | 균형잡힌 선택 | $15~20 |
| [Mailgun](https://www.mailgun.com/) | 개발자 친화적 | $35 |
| [Postmark](https://postmarkapp.com/) | 최고 전달률 | $15 |
| [Resend](https://resend.com/) | 모던 DX, React Email | $20 |

→ 상세 비교: [[01_API_이메일_서비스_비교]]

### 3.2 한국 이메일 서비스

→ 상세 정보: [[02_한국_이메일_서비스]]

| 도구 | 특징 | 월 비용 |

## 5. 용도별 추천

### 5.1 트랜잭션 이메일 (결제 확인서 등)

**최우선 고려사항**: 전달률, 속도, 안정성

| 순위 | 추천 | 이유 |
|------|------|------|
| 1 | **Postmark** | 전달률 최고 (93.8%), 트랜잭션 전문 |
| 2 | **AWS SES** | 최저가, 안정적 |
| 3 | **Resend** | 개발자 경험 최고 |

### 5.2 마케팅/뉴스레터

**최우선 고려사항**: 템플릿, 분석, 사용 편의성

| 순위 | 추천 | 이유 |
|------|------|------|
| 1 | **스티비** | 한글 UI, 쉬운 템플릿, A/B 테스트 |
| 2 | **SendGrid** | 마케팅 기능 포함 |
| 3 | **Mailchimp** | 종합 마케팅 플랫폼 |

### 5.3 둘 다 필요한 경우 (현재 상황)

**조합 전략 추천**:

```
┌────────────────────────────────────────────────────────────┐
│                    추천 조합 전략                            │
├──────────────┬─────────────────────────────────────────────┤
│ 비용 최적화   │ AWS SES (트랜잭션) + 스티비 (마케팅)          │
│              │ → 월 $1 + 스티비 요금                        │
├──────────────┼─────────────────────────────────────────────┤
│ 단일 서비스   │ SendGrid 또는 Resend                        │
│              │ → 트랜잭션 + 마케팅 모두 지원                  │
├──────────────┼─────────────────────────────────────────────┤
│ 노코드 선호   │ Make + 스티비                               │
│              │ → 코딩 없이 자동화                           │
└──────────────┴─────────────────────────────────────────────┘
```

## 6. 결론 및 최종 추천

### 월 1,000~10,000건, 트랜잭션+마케팅 기준

#### 🥇 1순위: AWS SES + 스티비 조합

```
비용: 월 ~$1 (AWS SES) + 스티비 요금
장점: 최저 비용, 한글 마케팅 도구
단점: 두 서비스 관리 필요
```

**적합한 경우**:
- 비용 효율이 가장 중요할 때
- 한글 마케팅 템플릿이 필요할 때
- AWS 사용 경험이 있을 때

#### 🥈 2순위: Resend (단일 서비스)

```
비용: 월 $20
장점: 최고의 개발자 경험, React Email
단점: 마케팅 기능 제한적
```

**적합한 경우**:
- 개발 편의성이 중요할 때
- React 사용에 익숙할 때
- 단일 서비스로 관리하고 싶을 때

#### 🥉 3순위: Make + 스티비 (노코드)

```
비용: 월 $9+ (Make) + 스티비 요금
장점: 코딩 불필요, 시각적 워크플로우
단점: 복잡한 로직에 한계
```

**적합한 경우**:
- 개발 리소스가 없을 때
- 빠르게 시작하고 싶을 때
- 간단한 자동화만 필요할 때
