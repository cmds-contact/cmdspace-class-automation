# 노코드 이메일 자동화 도구 비교

> Zapier, Make, n8n - 코딩 없이 이메일 자동화 워크플로우 구축하기

---

## 개요

노코드/로우코드 자동화 도구를 사용하면 프로그래밍 지식 없이도 이메일 자동화 워크플로우를 구축할 수 있습니다.

### 주요 사용 사례

- 구글 시트에 새 행 추가 시 이메일 발송
- 폼 제출 시 확인 이메일 자동 발송
- 결제 완료 시 영수증 이메일 발송
- CRM 이벤트 기반 후속 이메일 발송

---

## 1. Zapier

### 개요

| 항목 | 내용 |
|------|------|
| **설립** | 2011년 |
| **본사** | 미국 |
| **앱 연동** | **7,000개 이상** |
| **특징** | 가장 많은 앱 연동, 쉬운 사용성 |

### 가격 정책

| 플랜 | 태스크/월 | 월 비용 |
|------|----------|---------|
| Free | 100 | $0 |
| Starter | 750 | $19.99 |
| Professional | 2,000 | $49 |
| Team | 50,000 | $69/사용자 |
| Enterprise | 무제한 | 별도 문의 |

> **과금 방식**: 태스크(Task) 기준
> - 모든 액션이 태스크로 카운트
> - 2단계 워크플로우 = 2 태스크

### 장점

- **최대 앱 연동**: 7,000개 이상의 앱 지원
- **직관적 인터페이스**: 비개발자도 쉽게 사용
- **풍부한 템플릿**: 미리 만들어진 워크플로우 제공
- **안정성**: 오랜 운영 역사

### 단점

- **비용 증가 빠름**: 복잡한 워크플로우일수록 비용 급증
- **복잡한 로직 제한**: 조건문, 반복문 제한적
- **셀프호스팅 불가**: 클라우드 전용

### 이메일 자동화 예시

```
트리거: Google Sheets에 새 행 추가
  ↓
액션 1: 데이터 포맷팅
  ↓
액션 2: SendGrid로 이메일 발송
  ↓
액션 3: Slack 알림 전송
```

**위 워크플로우 = 4 태스크 소비**

### 지원하는 이메일 서비스

- Gmail / Google Workspace
- SendGrid
- Mailchimp
- Mailgun
- AWS SES (간접)
- Postmark
- 스티비

## 2. Make (구 Integromat)

### 개요

| 항목 | 내용 |
|------|------|
| **설립** | 2016년 |
| **본사** | 체코 |
| **앱 연동** | **약 2,000개** |
| **특징** | 비주얼 에디터, 복잡한 워크플로우 강점 |

### 가격 정책

| 플랜 | 작업/월 | 월 비용 |
|------|--------|---------|
| Free | 1,000 | $0 |
| Core | 10,000 | $9 |
| Pro | 10,000 | $16 |
| Teams | 10,000 | $29 |
| Enterprise | 커스텀 | 별도 문의 |

> **과금 방식**: 작업(Operation) 기준
> - 첫 모듈 = 1 작업
> - 라우터/에러 핸들러 = 0 작업
> - **Zapier보다 효율적인 과금**

### 장점

- **비주얼 워크플로우 에디터**: 직관적인 플로우 차트 방식
- **복잡한 로직 지원**: 라우터, 조건문, 반복문 강력
- **Zapier 대비 저렴**: 같은 워크플로우에서 비용 효율적
- **에러 핸들링**: 내장된 오류 처리 기능

### 단점

- **앱 연동 수 적음**: Zapier의 1/3 수준
- **학습 곡선**: 초기 학습 필요
- **셀프호스팅 불가**: 클라우드 전용

### 이메일 자동화 예시

```
┌─────────────┐
│  웹훅 트리거  │
└──────┬──────┘
       │
       ▼
┌─────────────┐
│  데이터 파싱  │
└──────┬──────┘
       │
       ▼
┌─────────────────────────────────┐
│           라우터                 │
├─────────────┬───────────────────┤
│ 조건: VIP   │ 조건: 일반 고객     │
│     ↓      │       ↓           │
│ VIP 템플릿  │  일반 템플릿        │
│ 이메일 발송  │  이메일 발송        │
└─────────────┴───────────────────┘
```

### 지원하는 이메일 서비스

- Gmail
- SendGrid
- Mailchimp
- Mailgun
- AWS SES
- Postmark
- SMTP (모든 서비스)

## 3. n8n

### 개요

| 항목 | 내용 |
|------|------|
| **설립** | 2019년 |
| **본사** | 독일 |
| **앱 연동** | **400개 이상** (네이티브) |
| **특징** | **오픈소스**, 셀프호스팅 가능 |

### 가격 정책

| 플랜 | 실행/월 | 월 비용 |
|------|--------|---------|
| **Self-hosted** | **무제한** | **$0** |
| Cloud Starter | 2,500 | $20 |
| Cloud Pro | 10,000 | $50 |
| Cloud Enterprise | 커스텀 | 별도 문의 |

> **과금 방식**: 실행(Execution) 기준
> - 2단계든 200단계든 **1회 실행 = 1 실행**
> - **가장 예측 가능한 비용 구조**

### 장점

- **완전 무료 가능**: 셀프호스팅 시 무제한 무료
- **데이터 프라이버시**: 자체 서버에서 실행
- **실행당 과금**: 복잡한 워크플로우도 비용 동일
- **코드 노드**: JavaScript/Python 코드 실행 가능
- **오픈소스**: 커스터마이징 가능

### 단점

- **앱 연동 수 적음**: 400개 (단, HTTP 요청으로 확장 가능)
- **셀프호스팅 기술 필요**: 서버 관리 필요
- **클라우드 비용 상대적 고가**: $20부터 시작

### 셀프호스팅 방법

**Docker로 설치**:
```bash
docker run -it --rm \
	--name n8n \
	-p 5678:5678 \
	-v ~/.n8n:/home/node/.n8n \
	n8nio/n8n
```

**Docker Compose**:
```yaml
version: '3'
services:
  n8n:
    image: n8nio/n8n
    ports:
      - "5678:5678"
    volumes:
      - ~/.n8n:/home/node/.n8n
    environment:
      - N8N_BASIC_AUTH_ACTIVE=true
      - N8N_BASIC_AUTH_USER=admin
      - N8N_BASIC_AUTH_PASSWORD=password
```

## 비용 시뮬레이션

### 시나리오: 월 1,000회 워크플로우 실행

각 워크플로우가 5단계라고 가정:

| 도구 | 계산 | 예상 비용 |
|------|------|----------|
| **Zapier** | 1,000 × 5 = 5,000 태스크 | ~$49/월 |
| **Make** | 1,000 × 5 = 5,000 작업 | ~$9/월 |
| **n8n (셀프)** | 무제한 | **$0** |
| **n8n (클라우드)** | 1,000 실행 | $20/월 |

## 이메일 자동화 워크플로우 예시

### 예시 1: 결제 완료 시 확인 이메일

```
트리거: Stripe 결제 완료 웹훅
  │
  ├─▶ 고객 정보 조회 (CRM)
  │
  ├─▶ 이메일 템플릿 생성
  │     - 고객명: {{customer.name}}
  │     - 결제 금액: {{payment.amount}}
  │     - 결제일: {{payment.date}}
  │
  └─▶ SendGrid/AWS SES로 발송
```

### 예시 2: 구글 시트 기반 대량 발송

```
트리거: 매일 오전 9시 (Schedule)
  │
  ├─▶ Google Sheets 읽기
  │     - 오늘 발송 대상 필터링
  │
  ├─▶ 반복 (Iterator)
  │     │
  │     └─▶ 각 행에 대해
  │           - 이름, 이메일 추출
  │           - 개인화 이메일 생성
  │           - 발송
  │
  └─▶ 발송 완료 로그 기록
```

### 예시 3: 폼 제출 → 자동 응답

```
트리거: Typeform/Google Forms 제출
  │
  ├─▶ 데이터 정규화
  │
  ├─▶ 조건 분기
  │     ├─ 문의 유형: 견적 요청
  │     │     └─▶ 견적서 템플릿 이메일 발송
  │     │
  │     └─ 문의 유형: 일반 문의
  │           └─▶ 자동 응답 이메일 발송
  │
  └─▶ Slack 알림 (담당자에게)
```

## 도구별 추천 상황

### Zapier 추천

- **비개발자** 또는 초보자
- **다양한 앱 연동** 필요 (7,000+ 앱)
- **간단한 워크플로우** (2-3단계)
- 빠른 시작 우선

### Make 추천

- **복잡한 비즈니스 로직** 필요
- 비용 효율 중요 (Zapier 대비)
- **시각적 워크플로우** 선호
- 에러 핸들링 중요

### n8n 추천

- **완전한 데이터 통제** 필요 (GDPR 등)
- **비용 $0** 원할 때 (셀프호스팅)
- **기술 팀** 보유
- 복잡한 자동화 + 코드 실행 필요
